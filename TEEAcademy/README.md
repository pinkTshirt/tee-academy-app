## Build status ‚Äî read this first

This project was generated in an environment with **no Android SDK, no Gradle
distribution, and no network access to Maven/Google's repositories**. That
means:

- The code has been verified **statically only**: every `.kt` file has
  balanced braces/parens, every Hilt `@Inject` constructor's dependencies
  resolve to something actually provided (DAOs ‚Üî `DatabaseModule`,
  repositories ‚Üî `RepositoryModule`), every Gradle version-catalog alias
  used in `build.gradle.kts` exists in `gradle/libs.versions.toml`, and
  every seed JSON file passes the same content-integrity checks the
  `SeedContentValidationTest` JUnit suite runs.
- It has **never been run through an actual Kotlin/Android compiler**. A
  first real build may still surface something this manual review missed
  (an actual compiler check is the only fully reliable verification).
- **There is no `gradlew` / Gradle wrapper committed to this repo.** The
  wrapper's jar file is a binary that has to be generated by a real Gradle
  install, which wasn't available when this project was generated.

## How to build a real APK

### Option A ‚Äî Android Studio (recommended, easiest)
1. Unzip this project and open the folder in Android Studio.
2. Let Gradle sync ‚Äî Android Studio will offer to generate the wrapper for
   you, or you can run `gradle wrapper` from its embedded terminal once.
3. `Build ‚Üí Build Bundle(s)/APK(s) ‚Üí Build APK(s)`.
4. Find the APK at `app/build/outputs/apk/debug/app-debug.apk`.

### Option B ‚Äî GitHub Actions (no local Android Studio needed)
This repo includes `.github/workflows/build-apk.yml`, which:
- Provisions a real Gradle binary via `gradle/actions/setup-gradle` (since
  this repo has no wrapper jar to bootstrap from).
- Generates a Gradle wrapper as part of the run (and uploads it as an
  artifact ‚Äî commit it back to the repo afterward if you want
  conventional `./gradlew` support going forward).
- Builds `assembleDebug` and runs the unit test suite.
- Uploads the resulting `app-debug.apk` as a downloadable build artifact.

Push this repo to GitHub (or just use `workflow_dispatch` to trigger it
manually from the Actions tab), then download the APK from the completed
run's "Artifacts" section.

### Option C ‚Äî Command line with a local Gradle/Android SDK install
```
gradle wrapper --gradle-version 8.7   # one-time, generates the missing wrapper
./gradlew assembleDebug
```
